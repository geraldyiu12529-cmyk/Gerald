2. The core problem — the "boilerplate leak"

Every scheduled task in your trading workspace is required by CLAUDE.md to run a Session Startup Protocol before doing any work: read the 8-step Methodology Prompt, Risk Rules, Data Sources, Coin Core, Trad Core, Memory.md, and the latest market brief. That is roughly 30–50K tokens of reading before the task even starts.

For the trade recommendation task, that is correct — the rec is a live trading decision and it must see the full methodology. For utility tasks it is pure waste:

  • us-close-snapshot just records what US markets did between 16:00 and 07:30 local. It doesn't need Risk Rules or the Trad Core to do that.
  • workspace-tidy archives old files and compacts memory. It doesn't need to know the 8-step framework to move a file from /Trade/ to /archive/.

Both tasks were reading the whole framework anyway, because the CLAUDE.md rule is global. That's the leak.

How big was the leak?

Each bar is the token count of a single scheduled run. For us-close-snapshot, about 40K of the 52K it burned was the framework it didn't need. For workspace-tidy, about 35K of 43K. The "useful work" portion (the actual snapshot / archival logic) is small — everything above it was boilerplate.

3. The fix — read-override prepend

The fix is a short block pasted at the top of each task's SKILL.md that says, in effect:

"You are a utility task. Skip the Session Startup Protocol. Here are the three specific files you need — read those and proceed."

Because the task prompt is read before the global CLAUDE.md rule triggers, this prepend wins. The global rule still applies to every other task — only these two utility tasks are carved out.

This is zero-risk: if it doesn't work, the task just does its old behaviour. A timestamped .bak-* copy of the original file sits next to each modified file until passive verification confirms nothing regressed.

6. Why the non-token fixes matter

A1 — positions-monitor (the monitoring gap)

Before today, nothing scheduled ever watched your open positions intraday. INTC, Gold, and QQQ were being held with no automated check on stop buffer, time-to-invalidation, earnings-window proximity, or portfolio-heat drift. If a thesis broke mid-session, only your own vigilance would catch it. The new cron runs every weekday at 09:00 UTC+8 and writes a flag report only if something is off — silent when OK.

E4 — shared catalysts cache

news-events, market-brief, and daily-trade-rec were each independently re-deriving the same catalyst calendar (Fed meetings, CPI prints, earnings dates, etc.). Three times the work, and with the added risk that one task's catalyst view could silently disagree with another's. Now news-events writes a canonical JSON cache, and brief + rec read it. Saves ~80–120K tokens/month and eliminates the cross-file-inconsistency class of bug.

consolidate-memory folder

That skill had been removed from references but its files remained on disk. Cosmetic cleanup, no behavioural effect — but if it reappears after a plugin update, that's an Anthropic-side default being pushed, and the fix is thumbs-down feedback, not another local delete.
