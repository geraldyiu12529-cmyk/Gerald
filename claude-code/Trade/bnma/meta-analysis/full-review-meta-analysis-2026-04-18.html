2. Inclusion & Exclusion Criteria

2.1 Paper-level inclusion (what counted as a source paper)

  • Bayesian Network Meta-Analysis runs — any paper in the workspace titled BNMA / System review with a posterior-mean + p_positive + p_beats_peers output structure. Inclusion requires either MCMC output or an analytical peer-dominance table with explicit RANK-1/BOTTOM-3 flags. Result: 4 included (P1 BNMA Mark 1, P2 ChatGPT Mark 2, P3 ChatGPT Mark 1 analytical, P4 BNMA Claude Mark 2).
  • Systematic reviews — any shortlist / candidate-pipeline paper that classifies variables as EFFECTIVE / INEFFECTIVE / MIXED with a grade (A–C or INCLUDE_UNGRADED / EXCLUDE / PARTIAL). Result: 8 included (5 from ChatGPT author, 3 from Claude author). Excluded: drafts without classifications, prompt files, pure outline docs.

2.2 Variable-level inclusion (what counted as a variable)

Five pre-specified criteria (from the system review skill):

  1. Peer-reviewed publication or named working paper with citable author.
  2. Replicated in at least one independent study — the raw original alone is not sufficient; post-publication replication required (HXZ 2020, JKP 2023, Goyal-Welch-Zafirov 2024 accepted as benchmarks).
  3. Mechanism-grounded — variable must have a documented economic mechanism, not just statistical fit.
  4. Independent of the existing V001–V028 registry (novelty test) — but retention of existing V-series is scored on the same criteria applied retrospectively.
  5. Real-time implementable — data available without look-ahead; documented vendor or public source.

2.3 Variable-level exclusion

  • Variables relying exclusively on in-sample fit (no OOS R² or live Sharpe from any replication).
  • Variables collapsed into composites already in the registry without additional mechanism (e.g., QMJ duplicates of GP/A were flagged PARTIAL, not independent).
  • Variables whose defining dataset no longer meets the real-time criterion (e.g., OptionMetrics-gated variables post-2020 vendor pricing changes were marked EXCLUDE in cl3).
  • Constructs known to have been arbitraged out with near-zero post-publication Sharpe in multiple replications (e.g., tail-risk composite post Bali-Cakici-Whitelaw 2024 audit).

2.4 Meta-analysis-level inclusion (what PL-NMA actually uses)

  • A variable is included in the PL-NMA if at least one paper produced an ordinal ranking signal — either a BNMA p_beats_peers value (BNMA runs) or a systematic-review score grade_numeric + 0.5·I[EFFECTIVE] − 0.5·I[INEFFECTIVE] + 0.3·sharpe_reported (reviews).
  • BNMA approx0 entries were omitted (not ranked) — the paper's own convention is that these are below-resolution, not equal.
  • Single-paper variables are kept but flagged — the PL-NMA posterior just shrinks them toward the N(0,1) prior.

3.2 Why Plackett-Luce, not direct Bayesian meta-regression?

Scale-commensurability problem. Each group reports its effect on a different scale: the S-group (structural) reports long–short Sharpe; the R-group (regime / risk-sizing) reports Sharpe improvement after applying a sizing overlay; the Overlay group reports conditional-Sharpe uplift. P4 (bnma-claude mark 2) §0 states explicitly that cross-group comparisons are not statistically valid on raw posterior means. A naïve pooled meta-regression on posterior medians would mistake scale differences for signal differences.

Plackett-Luce sidesteps this. It uses only within-paper orderings: whether variable i ranked above variable j in the same paper. This is invariant to the paper's choice of scale: if one paper reports Sharpe and another reports peer-dominance probability, the ordering is preserved. The latent strength θi is identified only up to the data's orderings, so the cross-group comparison is ordinal, not metric.

3.3 Model specification

For each ordered pair (i beats j) in the 810-pair extracted set, Bradley-Terry:

P(i beats j | θ) = σ(θi − θj)

Negative log-posterior:

−log p(θ | data) = Σpairs softplus(−(θw − θl)) + 0.5·Σ θi² / σ²prior

with σprior = 1.0. The log-posterior is globally log-concave (the softplus term is convex, the Gaussian prior is strictly convex). MAP via L-BFGS-B with analytic gradient; Laplace approximation via the Hessian at the MAP; 10,000 draws from the multivariate Normal approximation give posterior summaries and the all-vs-all dominance matrix.

3.4 Ordinal-input construction

  • BNMA papers (P1–P4): use p_beats_peers (joint-posterior peer-dominance probability), not posterior_median. Reason: the source BNMA authors themselves use p_beats_peers for the DEPLOY verdict, because peer-dominance reflects joint posterior competition, whereas marginal posterior mean is distorted by prior and scale. Using medians originally placed V010 (which is EXCLUDE) at rank 7 — fixed.
  • Systematic reviews (SR1–SR8): numeric score grade_numeric + 0.5·I[EFFECTIVE] − 0.5·I[INEFFECTIVE] + 0.3·sharpe_reported. Grade map: A=3, B=2, C=1, INCLUDE_UNGRADED=1.5, EXCLUDE=-1, PARTIAL=0.5.
  • Identifiability: N(0,1) prior on each θ is enough for symmetric identifiability without fixing a reference variable. Hessian min eigenvalue = 1.00 confirms strict log-concavity.

3.5 Sensitivity analyses

  • S1 — drop analytical-only papers (P2, P3 that have partial/no MCMC numerics). Tests whether analytical peer tables unduly influence top-5.
  • S2 — within-group BNMA fallback: re-fit PL-NMA using only within-group ranking pairs from BNMA papers (drops ~500 cross-group pairs). Tests how much of the top-5 ordering is a cross-scale artifact vs real ordering.
  • Robustness classification: a variable is robust-deploy if it ranks top-10 in primary ∧ S1 ∧ S2; primary-only if top-10 primary but >20 in S2 (signals a scale artifact); robust-exclude if bottom-10 in all three.

5.2 Discussion of analysing the 12 papers together

5.2.1 What the meta-analysis layers reveal

The cross-run BNMA synthesis (BNMA-meta-analysis-2026-04-18) already collapsed P1–P4 to a common variable registry and cleaned up heterogeneity flags. The PL-NMA adds two new insights:

  • The 3-variable spine holds even outside the BNMA sample. V009/V027/V028 keep top-10 positions when systematic-review variables (which P1–P4 didn't include) are added. V009 retains rank 1; V028 moves to rank 10 (S2 rank 3.35), V027 to rank 4 (S2 rank 14.7). The drop in V027's rank is a scale artifact — within R-group it's still rank 1.
  • The cross-group scale caveat is quantitatively confirmed. S2 (within-group) rearranges the top-10: variables that benefited from the Sharpe-scale bonus of S-group drop; variables that were under-represented relative to peers in R-group rise. The PL-NMA doesn't re-rank the within-group winners — those are stable — but it does reveal that the cross-group ordering is partly methodology, partly signal.
  • Single-paper candidates are correctly shrunk, not boosted. Variables that appear in only 1 systematic review (DealerGamma, PCTECH, CieslakPovala, OppInsider) end up at rank 5-15. That matches the intuition: high-grade but single-source evidence should be CONDITIONAL, not DEPLOY. The PL-NMA does this automatically via the weak prior.

5.2.2 What the papers don't agree on

  • V014 (BTC exchange netflows): P1 posterior 1.15 (huge); P2/P4 INDIST. DIVERGENCE·HETEROG flag across runs. The construct is splittable — order-flow leg works, vol-sort leg doesn't.
  • VRP: 6 reviews cite it; 3 EFFECTIVE, 1 C-grade MIXED, 1 EXCLUDE on decay. This is a post-publication decay disagreement, not a method disagreement.
  • GKXml: cg4 gave it raw Sharpe 2.45; cl3 EXCLUDE for V026 overlap; cg3 EXCLUDE for PIPE-02 paradigm overlap. High raw Sharpe / low marginal contribution: a classic factor zoo trap.

5.2.3 What the meta-analysis itself cannot answer

  • Joint portfolio performance. PL-NMA ranks individual variables. It does not answer 'which combination of the top-5 gives the best risk-adjusted portfolio' — that requires a joint covariance / crowding model, which the workspace does not currently run.
  • Regime-conditionality. The ranking is unconditional. V027's value as a regime gate is by definition conditional. A regime-conditional extension would require per-regime ordinal inputs, which the 12 papers don't always provide.
  • Transaction-cost bite. All Sharpes entering the SR score are gross or lightly-netted. The registry already has paradigm-level haircuts (PCTECH 0.55 gross → 0.30 net), but those haircuts aren't flowed through PL-NMA. Treat the ranking as alpha-space, not P&L-space.

6. Discussion — grouping and individual variables

6.1 Should the S / T / R / Overlay / Stocks grouping be retained?

Recommendation: KEEP the grouping, do NOT use it as a hard quota; DO use it as a commensurability scope.

Reasoning: The grouping serves three distinct purposes, only one of which is statistical.
  • Purpose 1 — mechanism identification (keep). Grouping by S/T/R labels the role a variable plays (Sharpe-generator, regime-filter, risk-sizing scalar, overlay). Mixing these in a sizing engine would double-count (two Sharpe-generators get full weight; two regime filters need a conjunction rule, not addition).
  • Purpose 2 — commensurability scope (keep for synthesis). P4 §0 is correct: cross-group posterior comparisons are not valid on raw means. For within-group synthesis the mean is comparable; for cross-group a ranking model (PL-NMA) is required.
  • Purpose 3 — allocation weighting (drop). Requiring each group to have ≥N DEPLOYs forces CONDITIONAL variables to be promoted (e.g., V026, V018) just to fill R / Overlay rows. The PL-NMA shows this is wasted sizing — V026 is peer-dominated by V009 in every single run.

Operational recommendation: keep S/T/R as paradigm labels and as the commensurability scope for synthesis. Do not enforce 'each group must have ≥N DEPLOYs'. Let the empty R-group sit empty if the evidence says only V027 makes it.
